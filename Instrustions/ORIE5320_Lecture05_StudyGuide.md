### 第 1 页：Single Dimensional Optimization（续）

**原文翻译**：

一维单峰函数的优化框架。在区间 $[a^k, b^k]$ 中取两个试探点 $\lambda^k$（左侧）和 $\rho^k$（右侧）。

**情况 1**：当 $f(\lambda^k) \leq f(\rho^k)$ 时，更新区间为：
- $a^{k+1} = a^k$
- $b^{k+1} = \rho^k$

新区间为 $[a^{k+1}, b^{k+1}]$。

**情况 2**：当 $f(\rho^k) < f(\lambda^k)$ 时，更新区间为：
- $a^{k+1} = \lambda^k$
- $b^{k+1} = b^k$

新区间为 $[a^{k+1}, b^{k+1}]$。

**精讲内容**：

这一页展示的是一维优化最基本的框架——**区间缩小法**（Interval Reduction Method）。核心思想极其简洁：我们维护一个"不确定区间" $[a^k, b^k]$，确保最小值始终在这个区间内，然后通过每一步迭代不断缩小这个区间。

为什么这个方法能成立？它依赖一个关键假设：$f$ 是**单峰函数**（unimodal function）。单峰函数的定义是：在区间上存在唯一的极小值点 $x^*$，且 $f$ 在 $x^*$ 左侧单调递减，在 $x^*$ 右侧单调递增。这意味着函数图形呈现一个"碗"的形状。

在这个假设下，我们在区间内取两个试探点 $\lambda^k < \rho^k$，然后比较它们的函数值：

- 如果 $f(\lambda^k) \leq f(\rho^k)$：这意味着什么？由于函数是单峰的，$\rho^k$ 处的函数值更大，说明最小值点不可能在 $\rho^k$ 的右边（否则函数从 $\lambda^k$ 到 $\rho^k$ 应该一直下降）。所以我们可以安全地把右端点从 $b^k$ 缩小到 $\rho^k$。

- 如果 $f(\rho^k) < f(\lambda^k)$：对称地，最小值点不可能在 $\lambda^k$ 的左边。所以把左端点从 $a^k$ 推进到 $\lambda^k$。

这个逻辑本质上是一种**排除法**——每次比较都能排除区间的一部分，保证最小值仍然在剩余区间内。

从 Lec 04 的梯度下降视角来看，这里的一维函数可以理解为沿梯度方向的"切面"函数。也就是说，我们已经知道了下降方向（负梯度），现在要在这个方向上找到最佳步长。这正是 Lec 06 将要展示的 line search 应用。

值得注意的是，这个框架对试探点的具体位置没有做任何要求——$\lambda^k$ 和 $\rho^k$ 可以是区间内的任意两点（只要 $\lambda^k < \rho^k$）。不同的放置策略产生不同的算法：二分搜索将它们放在中点两侧，黄金分割搜索则有一种更巧妙的放法。

**教授口吻讲解**：

同学你看，一维优化的起点其实很朴素——手里有一个碗形的函数，不知道碗底在哪，怎么办？伸两根手指进碗里试试呗！左手指摸到的地方更低，说明碗底在左边；右手指更低，碗底就在右边。然后把碗不需要的那半边盖上，继续摸。每一轮都能排除掉一块区域，这就是区间缩小法的全部精髓。

但这里有个前提假设——函数必须是单峰的。如果函数有多个局部最小值，这个方法就不靠谱了，因为"左边低所以碗底在左边"这个推理链条会断裂。这也是为什么我们后面讲凸优化特别重要——凸函数天然就是单峰的，全局最小值唯一。

### 第 2 页：Bisection Search for Single-Dimensional Optimization

**原文翻译**：

[图示] 数轴上标出区间 $[a^k, b^k]$，中点为 $(a^k + b^k)/2$。$\lambda^k$ 在中点左侧距离 $\varepsilon$ 处，$\rho^k$ 在中点右侧距离 $\varepsilon$ 处。

算法步骤：

(1) 选择初始不确定区间 $(a^1, b^1)$，选择停止容差 $\theta$，设迭代计数 $k = 1$。

(2) 设定：

$$\lambda^k = \frac{a^k + b^k}{2} - \varepsilon, \qquad \rho^k = \frac{a^k + b^k}{2} + \varepsilon$$

(3) 如果 $f(\lambda^k) \leq f(\rho^k)$，则 $a^{k+1} = a^k$，$b^{k+1} = \rho^k$。
如果 $f(\lambda^k) > f(\rho^k)$，则 $a^{k+1} = \lambda^k$，$b^{k+1} = b^k$。

(4) 如果 $b^{k+1} - a^{k+1} \leq \theta$ 则停止。否则 $k$ 加 1，返回步骤 2。

**精讲内容**：

二分搜索是区间缩小法的最自然实现。它的核心思想是：把两个试探点放在区间中点的两侧，各偏移一个很小的距离 $\varepsilon$。

让我们分析每一步的区间缩小比例。假设当前区间长度为 $L^k = b^k - a^k$：

- 情况 1（$f(\lambda^k) \leq f(\rho^k)$）：新区间为 $[a^k, \rho^k]$，长度为 $\rho^k - a^k = \frac{a^k + b^k}{2} + \varepsilon - a^k = \frac{L^k}{2} + \varepsilon$
- 情况 2（$f(\rho^k) < f(\lambda^k)$）：新区间为 $[\lambda^k, b^k]$，长度为 $b^k - \lambda^k = b^k - \frac{a^k + b^k}{2} + \varepsilon = \frac{L^k}{2} + \varepsilon$

所以每步区间缩小到 $\frac{L^k}{2} + \varepsilon$。当 $\varepsilon$ 很小时，这近似为 $\frac{L^k}{2}$，即每步减半——这就是"二分"名称的来源。

$\varepsilon$ 的角色非常微妙。如果 $\varepsilon = 0$，两个试探点重合在中点，我们就无法判断最小值在左还是右。$\varepsilon$ 必须大于零，但又不能太大——太大会降低每步的缩小效率。通常取 $\varepsilon$ 远小于初始区间长度，比如 $\varepsilon = 10^{-6}$。

**收敛速度分析**：经过 $k$ 次迭代后，区间长度约为 $L^0 \cdot (\frac{1}{2})^k$（忽略 $\varepsilon$）。要达到精度 $\theta$，需要 $k \approx \log_2(L^0/\theta)$ 次迭代。每次迭代需要两次函数评估（计算 $f(\lambda^k)$ 和 $f(\rho^k)$），所以总共需要约 $2\log_2(L^0/\theta)$ 次函数评估。

这个方法的问题在于：每次迭代的两次函数评估是"一次性"的——下一次迭代完全重新计算，之前的计算结果无法复用。这就引出了一个自然的问题：能不能做到每步只需一次新的函数评估？

**教授口吻讲解**：

二分搜索你应该不陌生——这是搜索算法里最经典的思想。猜数字游戏对吧？1 到 100 之间猜一个数，每次告诉你高了还是低了，你总是猜中间值。这里也一样，只不过不是"高了低了"，而是"左边低还是右边低"。

但注意一个关键区别：猜数字时每次只需一次查询（"是不是 50？""高了"），而二分搜索一维优化每次需要**两次**函数评估。为什么？因为我们不是在比较和一个已知值的关系，而是在比较两个未知点的函数值。这两次评估意味着效率有提升空间——接下来的黄金分割法就是来解决这个问题的。

### 第 3 页：二分搜索示例与动机

**原文翻译**：

- 用二分搜索最小化 $x^2 + 2x$ 在 $[-3, 5]$ 上的值。

- 二分搜索每次迭代需要两次函数评估：$f(\lambda^k)$ 和 $f(\rho^k)$。

- 我们能否在每次迭代中只进行一次函数评估？

**精讲内容**：

这一页通过一个具体例子引出核心动机。$f(x) = x^2 + 2x$ 是一个简单的二次函数，其最小值点在 $x^* = -1$（通过 $f'(x) = 2x + 2 = 0$ 可得）。当然，对于二次函数我们可以直接求导解方程，不需要搜索。但这个例子的目的是演示二分搜索的过程，并让我们体会到"两次评估"的浪费。

让我们手算第一步。初始区间 $[-3, 5]$，中点为 $1$，取 $\varepsilon = 0.1$：
- $\lambda^1 = 1 - 0.1 = 0.9$，$f(0.9) = 0.81 + 1.8 = 2.61$
- $\rho^1 = 1 + 0.1 = 1.1$，$f(1.1) = 1.21 + 2.2 = 3.41$

因为 $f(\lambda^1) < f(\rho^1)$，新区间为 $[-3, 1.1]$。

接下来第二步，中点为 $(-3 + 1.1)/2 = -0.95$：
- $\lambda^2 = -0.95 - 0.1 = -1.05$，$f(-1.05) = 1.1025 - 2.1 = -0.9975$
- $\rho^2 = -0.95 + 0.1 = -0.85$，$f(-0.85) = 0.7225 - 1.7 = -0.9775$

因为 $f(\lambda^2) < f(\rho^2)$，新区间为 $[-3, -0.85]$。

注意，$\lambda^1 = 0.9$ 和 $\rho^1 = 1.1$ 的函数值在第二步中完全没有被使用。这就是"浪费"的来源——每步计算的两个值在下一步都被丢弃了。

这个观察自然引出了黄金分割搜索的核心动机：**能不能设计试探点的位置，使得上一步的某个试探点恰好可以在下一步中复用？** 如果可以，每步就只需要一次新的函数评估。

**教授口吻讲解**：

$x^2 + 2x$ 这个例子太简单了，一眼就能看出最小值在 $-1$。但它作为教学工具非常好——让你亲手体验"每步算两次真的很浪费"。你想想，科学计算中函数评估可能非常昂贵——一次函数评估可能意味着运行一个复杂的仿真或者训练一个子模型。如果能把每步的函数评估从两次降到一次，效率翻倍，这在实际应用中价值巨大。

所以问题就变成了：能不能让上一步的"废料"变成下一步的"原料"？这个问题的答案——黄金分割——将引出数学中最优美的常数之一。

### 第 4 页：Golden Section Search for One-Dimensional Optimization

**原文翻译**：

- 我们能否设置不确定区间中的左右试探点，使得 $\lambda^{k+1}$ 匹配 $\rho^k$，或者 $\rho^{k+1}$ 匹配 $\lambda^k$？

[图示——情况 $\star$] 区间 $[a^k, b^k]$ 上标出 $\lambda^k$ 和 $\rho^k$。更新后新区间 $[a^{k+1}, b^{k+1}] = [a^k, \rho^k]$（情况 1 发生时），在新区间中 $\lambda^{k+1}$ 已知，只需计算 $f(\rho^{k+1})$。

[图示——情况 $\star\star$] 同一结构。更新后新区间 $[a^{k+1}, b^{k+1}] = [\lambda^k, b^k]$（情况 2 发生时），在新区间中 $\rho^{k+1}$ 已知，只需计算 $f(\lambda^{k+1})$。

- 除了这种对齐条件之外，我们还要求不确定区间在每次迭代中以固定比例 $\beta$ 缩小：

$$b^{k+1} - a^{k+1} = \beta \cdot (b^k - a^k)$$

**精讲内容**：

这一页是黄金分割搜索的**设计思想**页——它不是直接给出算法，而是提出了两个约束条件，然后问：满足这两个条件的试探点位置是否存在？

**约束 1：试探点复用**。在情况 $\star$（$f(\lambda^k) \leq f(\rho^k)$，新区间为 $[a^k, \rho^k]$）中，我们希望上一步的 $\rho^k$ 变成新区间中的 $\lambda^{k+1}$——这样 $f(\lambda^{k+1}) = f(\rho^k)$ 已知，只需计算一个新点 $\rho^{k+1}$。对称地，在情况 $\star\star$ 中，$\lambda^k$ 应变成 $\rho^{k+1}$。

**约束 2：等比缩小**。区间长度以固定比例 $\beta$ 缩小。这是一个对称性要求——无论发生哪种情况，缩小的比例都一样。这保证了算法的收敛速度是可预测的。

这两个约束的组合将唯一确定 $\beta$ 的值。让我们直观理解为什么：约束 1 规定了试探点在新区间中的"相对位置"，约束 2 规定了新区间的"绝对大小"。两者结合，试探点的位置就被完全锁定了。

从工程角度看，这是一种非常优雅的设计方法论——先声明你想要什么性质，然后推导出满足这些性质的参数值。这种"约束驱动设计"在优化理论中非常普遍——你在 Lec 06 看到的惩罚函数法也是类似的思路：把约束转化为目标函数中的项。

**教授口吻讲解**：

这一页的思维方式特别值得学习。不是"我猜一个 $\beta$ 试试看"，而是"我先列出我想要的性质，然后看看什么样的 $\beta$ 满足"。这就是数学家做事的方式——先写方程，再求解。

试探点复用的想法很自然：我已经花了计算资源算出了 $f(\rho^k)$，下一步能不能不扔掉它？这就好比你做实验收集了数据，换个角度再分析一遍，而不是重新做实验。在计算昂贵的场景下，这种"回收利用"思想能节省一半的计算量。

### 第 5 页：黄金分割比推导（情况 $\star$）

**原文翻译**：

- 如果情况 $\star$ 发生，区间缩小 $\beta$ 倍：

$$b^{k+1} - a^{k+1} = \beta(b^k - a^k) \implies b^k - \lambda^k = \beta(b^k - a^k)$$

$$\implies \lambda^k = \beta \cdot a^k + (1 - \beta) \cdot b^k$$

- 如果情况 $\star\star$ 发生，区间缩小 $\beta$ 倍：

$$b^{k+1} - a^{k+1} = \beta(b^k - a^k) \implies \rho^k - a^k = \beta(b^k - a^k)$$

$$\implies \rho^k = (1 - \beta) \cdot a^k + \beta \cdot b^k$$

- 如果情况 $\star$ 发生，确保 $\lambda^{k+1}$ 匹配 $\rho^k$：

$$\rho^k = \lambda^{k+1} = \beta \cdot a^{k+1} + (1 - \beta) \cdot b^{k+1} = \beta \cdot a^k + (1 - \beta) \cdot \rho^k \qquad (\ddagger)$$

将 $\rho^k = (1-\beta) a^k + \beta b^k$ 代入条件 $(\ddagger)$：

$$(1 - \beta) a^k + \beta b^k = \beta^2 a^k + (\beta - \beta^2) b^k + (1 - \beta) b^k$$

$$(1 - \beta)(a^k - b^k) = \beta^2(a^k - b^k) \implies 1 - \beta = \beta^2 \implies \beta^2 + \beta - 1 = 0$$

$$\beta = \frac{-1 + \sqrt{5}}{2}$$

**精讲内容**：

这一页是本讲的数学核心——推导出黄金分割比 $\beta = \frac{\sqrt{5} - 1}{2} \approx 0.618$。让我们逐步拆解推导过程。

**第一步：从缩小比例确定试探点位置。**

在情况 $\star$ 中，新区间是 $[a^k, \rho^k]$（即 $a^{k+1} = a^k$，$b^{k+1} = \rho^k$），长度为 $\rho^k - a^k$。等比缩小条件给出：

$$\rho^k - a^k = \beta(b^k - a^k)$$

这意味着 $\rho^k$ 把区间 $[a^k, b^k]$ 按 $\beta : (1-\beta)$ 的比例分割。类似地，$\lambda^k$ 把区间按 $(1-\beta) : \beta$ 分割。注意 $\lambda^k$ 和 $\rho^k$ 关于区间中点对称——这是等比缩小条件的自然结果。

**第二步：复用条件。**

情况 $\star$ 发生后，新区间 $[a^{k+1}, b^{k+1}] = [a^k, \rho^k]$。我们要求新区间中的 $\lambda^{k+1}$ 恰好等于原区间中的某个试探点。由于 $\lambda^{k+1}$ 应该将新区间按 $(1-\beta) : \beta$ 从右端分割：

$$\lambda^{k+1} = \beta \cdot a^{k+1} + (1-\beta) \cdot b^{k+1} = \beta \cdot a^k + (1-\beta) \cdot \rho^k$$

而我们希望 $\lambda^{k+1} = \rho^k$（原来的右试探点变成新的左试探点）。但这里 slides 中实际写的是 $\rho^k = \lambda^{k+1}$——注意这一页中间的推导实际是把"下一步的 $\lambda$"用当前的 $a^{k+1} = a^k$ 和 $b^{k+1} = \rho^k$ 表达出来。

**第三步：推导方程。**

将所有表达式代入，最终化简为 $1 - \beta = \beta^2$，即 $\beta^2 + \beta - 1 = 0$。用求根公式：

$$\beta = \frac{-1 \pm \sqrt{1 + 4}}{2} = \frac{-1 \pm \sqrt{5}}{2}$$

取正根（因为 $\beta$ 是缩小比例，必须 $0 < \beta < 1$）：$\beta = \frac{\sqrt{5} - 1}{2} \approx 0.6180$。

这个数就是著名的**黄金比例的共轭值**。黄金比例 $\phi = \frac{1+\sqrt{5}}{2} \approx 1.618$，而 $\beta = \phi - 1 = 1/\phi$。黄金比例满足 $\phi^2 = \phi + 1$，等价于 $\beta$ 满足 $\beta^2 + \beta = 1$。这个自相似性质（整体的比例等于局部的比例）正是黄金分割搜索能够复用试探点的数学根源。

**教授口吻讲解**：

这个推导过程看起来有点长，但核心思想很简单：我们列了两个方程（等比缩小 + 试探点复用），两个方程包含一个未知数 $\beta$，解出来就完了。答案恰好是黄金比例——这不是巧合，而是黄金比例的自相似性质的必然结果。

$\beta \approx 0.618$ 意味着什么？每步区间缩小到原来的 $61.8\%$，比二分搜索的 $50\%$ 差一些。但记住：二分搜索每步需要**两次**函数评估，黄金分割只需要**一次**。如果公平比较（按函数评估次数而非迭代次数），黄金分割其实更高效。具体来说，达到同样精度，二分搜索需要 $2\log_2(L^0/\theta)$ 次评估，黄金分割需要 $\log_\beta(L^0/\theta) \approx 2.08\log_2(L^0/\theta)$ 次，差距很小，但黄金分割的实现更简洁，且不需要选择 $\varepsilon$ 参数。

### 第 6 页：黄金分割比推导（情况 $\star\star$）

**原文翻译**：

- 如果情况 $\star\star$ 发生，确保 $\rho^{k+1}$ 匹配 $\lambda^k$：

$$\lambda^k = \rho^{k+1} = (1 - \beta) \cdot a^{k+1} + \beta \cdot b^{k+1} = (1 - \beta) \cdot \lambda^k + \beta \cdot b^k \qquad (\ddagger)$$

记住：

$$\lambda^k = \beta \cdot a^k + (1 - \beta) \cdot b^k, \qquad \rho^k = (1 - \beta) \cdot a^k + \beta \cdot b^k$$

代入条件 $(\ddagger)$：

$$\beta \cdot a^k + (1 - \beta) \cdot b^k = (1 - \beta) \cdot a^k + (\beta - \beta^2) \cdot a^k + \beta^2 \cdot b^k$$

$$(1 - \beta)(b^k - a^k) = \beta^2(b^k - a^k) \implies 1 - \beta = \beta^2 \implies \beta^2 + \beta - 1 = 0$$

$$\beta = \frac{-1 + \sqrt{5}}{2}$$

**精讲内容**：

这一页验证了对称情况——当情况 $\star\star$ 发生时，是否能得到相同的 $\beta$ 值？答案是肯定的。

在情况 $\star\star$ 中，$f(\rho^k) < f(\lambda^k)$，所以新区间是 $[\lambda^k, b^k]$（$a^{k+1} = \lambda^k$，$b^{k+1} = b^k$）。我们要求新区间中的 $\rho^{k+1}$ 恰好等于原区间中的 $\lambda^k$。但 $\lambda^k = a^{k+1}$ 恰好是新区间的左端点，所以我们实际希望的是 $\rho^{k+1}$ 在新区间中的位置与 $\lambda^k$ 的位置关系正确——等一下，让我重新理解：

实际上，在情况 $\star\star$ 中：$a^{k+1} = \lambda^k$，$b^{k+1} = b^k$。新区间中 $\rho^{k+1}$ 应等于旧的 $\rho^k$（因为旧的 $\rho^k$ 仍在新区间内，且位置恰当）。slides 这里的推导是从 $\lambda^k = \rho^{k+1}$ 出发——但更准确理解是通过代数推导验证一致性。

无论如何，最终的方程完全一样：$1 - \beta = \beta^2$，解得 $\beta = \frac{\sqrt{5}-1}{2}$。这说明了系统的**对称性**——无论哪种情况发生，缩小比例和试探点复用机制都完全一致。这种对称性是黄金分割搜索的美学所在。

从更深的数学视角看，$\beta^2 + \beta = 1$ 这个方程可以改写为 $\beta = 1 - \beta^2 = (1-\beta)(1+\beta)$。由于 $\beta = 1/(1+\beta)$（黄金比例的递归性质），整个区间的分割结构在不同尺度上是自相似的。这种分形般的结构也是黄金比例在自然界频繁出现的数学原因——向日葵种子的排列、鹦鹉螺壳的螺旋都遵循同样的比例。

**教授口吻讲解**：

有同学可能会问：为什么要验证两种情况？因为我们需要确保算法的对称性——无论 $f(\lambda^k)$ 更小还是 $f(\rho^k)$ 更小，同一个 $\beta$ 值都能工作。如果两种情况给出不同的 $\beta$，那这个算法就设计不出来了。幸运的是，两种情况给出的方程完全一样。这就像一个硬币的两面——看起来不同，但都是同一枚硬币。

黄金比例 $\approx 0.618$ 在数学里有太多神奇的性质了。我告诉你一个：$0.618^2 = 0.382 = 1 - 0.618$。也就是说，如果你把一根绳子按黄金比例分成长和短两段，长段再按同样比例分，短段的长度恰好等于原来的短段。这种"缩放后自相似"的性质，正是黄金分割搜索能复用试探点的本质原因。

### 第 7 页：Golden Section Search 算法与数值示例

**原文翻译**：

[图示] 三层嵌套区间的数值示例：
- 第一层：$[0, 10]$，试探点在 $3.82$ 和 $6.18$
- 第二层：$[3.82, 10]$，试探点在 $6.18$ 和 $7.64$
- 第三层：$[3.82, 7.64]$，试探点在 $5.28$ 和 $6.18$

每层区间宽度缩减为上一层的 $\beta \approx 0.618$ 倍。

算法步骤：

(1) 选择 $(a^1, b^1)$，选择停止容差 $\theta$，设 $k = 1$。

(2) 设定 $\lambda^k = \beta \cdot a^k + (1-\beta) \cdot b^k$，$\rho^k = (1-\beta) \cdot a^k + \beta \cdot b^k$，其中 $\beta = \frac{-1+\sqrt{5}}{2}$。

(3) 如果 $f(\lambda^k) \leq f(\rho^k)$，则 $a^{k+1} = a^k$，$b^{k+1} = \rho^k$。
如果 $f(\lambda^k) > f(\rho^k)$，则 $a^{k+1} = \lambda^k$，$b^{k+1} = b^k$。

(4) 如果 $b^{k+1} - a^{k+1} \leq \theta$ 则停止。否则 $k$ 加 1，返回步骤 2。

- 不要忘记从 $f(\rho^{k-1})$ 借用 $f(\lambda^k)$ 或从 $f(\lambda^{k-1})$ 借用 $f(\rho^k)$。

**精讲内容**：

这一页将前面的理论推导整合为完整的算法，并用数值示例直观展示区间缩小的过程。

**数值示例解读**：以 $[0, 10]$ 为例，$\beta \approx 0.618$：
- $\lambda^1 = 0.618 \times 0 + 0.382 \times 10 = 3.82$
- $\rho^1 = 0.382 \times 0 + 0.618 \times 10 = 6.18$
- 区间长度 = 10

假设 $f(3.82) > f(6.18)$（最小值在右侧），新区间为 $[3.82, 10]$，长度 = $6.18 \approx 10 \times 0.618$。

在新区间 $[3.82, 10]$ 中：
- $\rho^2 = 0.382 \times 3.82 + 0.618 \times 10 = 7.64$
- $\lambda^2 = 0.618 \times 3.82 + 0.382 \times 10 = 6.18$ —— **恰好等于上一步的 $\rho^1$**！

所以 $f(\lambda^2) = f(6.18)$ 已经在上一步计算过了，不需要重新计算。这就是试探点复用的实际效果。

**最后一行的提醒至关重要**：算法的正确实现必须记住前一步的函数值。如果忘记复用，每步仍然需要两次函数评估，黄金分割搜索就退化为一个缩小比例更差的二分搜索了。

**收敛速度比较**：

| 方法 | 每步函数评估 | 每步缩小比 | $k$ 步后区间 | 达到精度 $\theta$ 所需评估次数 |
|------|------------|-----------|-------------|--------------------------|
| 二分搜索 | 2 | $\approx 0.5$ | $L_0 \cdot 0.5^k$ | $\approx 2\log_2(L_0/\theta)$ |
| 黄金分割 | 1（首步2） | $0.618$ | $L_0 \cdot 0.618^k$ | $\approx 2.08\log_2(L_0/\theta)$ |

按函数评估次数计算，两种方法效率相当。但黄金分割搜索的优势在于：(1) 不需要选择 $\varepsilon$ 参数；(2) 实现更简洁；(3) 区间缩小率严格固定。

**教授口吻讲解**：

看那个数值例子——$6.18$ 这个数在第一步是 $\rho^1$，在第二步变成了 $\lambda^2$。一个数换了个名字就被"回收利用"了，省了一次昂贵的函数评估。在机器学习中，一次函数评估可能意味着一次完整的前向传播、一次模型评估、甚至一次超参数搜索的完整训练——每省一次都是真金白银。

最后那行"不要忘记借用"，是教授在提醒你写代码时最容易犯的错——逻辑上明白要复用，但实现时忘了把上一步的值传下来。这种细节决定了你的代码是教科书级别的还是"能跑但没效率"的。

### 第 8 页：利用导数的单维优化

**原文翻译**：

- 这些单维优化算法只使用函数值评估，不使用函数的导数。如果我们能获取 $f$ 的导数，可以更高效。

[图示——左图] 一维凸函数曲线，在区间 $[a^k, b^k]$ 中点 $(a^k+b^k)/2$ 处求导。

当 $f'((a^k+b^k)/2) \leq 0$ 时（中点处导数为负，函数仍在递减），最小值在中点右侧：
- $a^{k+1} = (a^k+b^k)/2$
- $b^{k+1} = b^k$

新区间为右半部分。

[图示——右图] 同一结构。

当 $f'((a^k+b^k)/2) > 0$ 时（中点处导数为正，函数在递增），最小值在中点左侧：
- $a^{k+1} = a^k$
- $b^{k+1} = (a^k+b^k)/2$

新区间为左半部分。

**精讲内容**：

这一页引入了第三种一维优化方法——**基于导数的搜索**。核心思想极其直接：在中点处计算导数，导数的符号告诉我们最小值在左还是右。

为什么导数能给出方向信息？对于单峰函数：
- $f'(x) < 0$：函数在 $x$ 处递减，说明最小值在 $x$ 的右边
- $f'(x) > 0$：函数在 $x$ 处递增，说明最小值在 $x$ 的左边
- $f'(x) = 0$：$x$ 就是最小值点

这比之前的方法有一个巨大优势：**每步只需一次导数评估，就能将区间严格减半**。不需要两个试探点，不需要 $\varepsilon$ 参数，也不需要黄金比例。

**与二分搜索的对比**：
- 二分搜索：每步两次函数评估，区间近似减半
- 基于导数：每步一次导数评估，区间精确减半

如果导数评估和函数评估的代价相当，那么基于导数的方法效率翻倍。但关键前提是：**必须能计算导数**。在很多实际场景中（黑盒优化、仿真优化），函数可能没有解析导数，只能通过函数值评估来操作。

从课程脉络看，这一页建立了"有导数 vs 无导数"的方法论对比：
- **无导数方法**（零阶方法）：二分搜索、黄金分割搜索——只需函数值
- **一阶方法**：基于导数的搜索——需要梯度
- 这个分类在多维优化中同样存在：梯度下降（一阶）vs 无梯度优化（零阶）vs 牛顿法（二阶）

这也预示了 Lec 06 中 line search 的两种实现：可以用黄金分割搜索来做 line search（不需要一维导数），也可以用导数方法（如果沿搜索方向的方向导数容易计算）。

**教授口吻讲解**：

这一页的信息量不大但观念很重要。之前两种方法——二分搜索和黄金分割——都是"盲人摸象"式的：只知道两个点的函数值高低，不知道函数的"走势"。导数就像给了你一个指南针——不用比较两个点，直接看中点处的"坡度方向"就知道碗底在哪边了。

但天下没有免费的午餐。导数的计算是有代价的——在深度学习中，反向传播计算梯度的成本大约是前向传播的 2-3 倍。所以"导数免费"这个假设不总是成立的。什么时候用什么方法，取决于你的信息条件——这和我们做工程优化是一样的道理：信息越多，效率越高。

下一讲我们会看到，在梯度下降的步长选择中，这些一维搜索方法是怎么作为"子程序"被调用的。所以今天学的不是孤立的知识，而是明天要用的工具。

## 核心收获

**收获一：一维优化是多维优化的子程序——步长选择的数学基础**

本讲看似讨论一维问题，实际是为梯度下降的步长选择（line search）提供工具。当你在多维空间中确定了下降方向后，沿该方向找到最优步长本质上是一个一维优化问题。二分搜索和黄金分割搜索提供了不需要导数的求解方法，而基于导数的搜索在可微条件下效率更高。

**收获二：试探点复用是效率提升的核心——从二分搜索到黄金分割的演进逻辑**

二分搜索每步两次函数评估，黄金分割搜索通过巧妙的试探点布局实现了每步仅一次新评估。这种"回收利用"的思想在优化算法设计中非常普遍——后续的动量法（momentum）、Adam 优化器等都在不同层面上实现了信息复用。黄金比例 $\beta = \frac{\sqrt{5}-1}{2}$ 的出现不是偶然，而是等比缩小和自相似性条件的唯一解。

**收获三：信息条件决定算法效率——零阶 vs 一阶方法的根本区别**

无导数方法（零阶）和有导数方法（一阶）的效率差距在一维问题中就已经清晰可见。基于导数的搜索用一次导数评估实现精确二分，比黄金分割更高效。这个"信息越多效率越高"的原则贯穿整个优化课程——一阶方法（梯度下降）优于零阶，二阶方法（牛顿法）在条件好时又优于一阶。
