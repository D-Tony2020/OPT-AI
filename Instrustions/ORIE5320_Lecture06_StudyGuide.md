### 第 1 页：Using Single-Dimensional Optimization to Find Best Step Size During Gradient Descent

**原文翻译**：

- 我们在用梯度下降最小化 $f: \mathbb{R}^n \to \mathbb{R}$。在梯度下降的第 $k$ 次迭代中，当前位于点 $x^k$。

- 计算 $\nabla f(x^k)$，想沿负梯度方向移动步长 $\alpha$。

[图示] 从点 $x^k$ 出发，沿负梯度方向 $\nabla f(x^k)$（红色虚线箭头）移动步长 $\alpha$，到达点 $x^k - \alpha \nabla f(x^k)$。

- 选择步长 $\alpha$ 使得到达的下一个点的函数值尽可能小：

$$\min_{\alpha \geq 0} f\left(x^k - \alpha \, \nabla f(x^k)\right)$$

其中 $x^k$ 已知，$\nabla f(x^k)$ 已知。

- 这是一个单维优化问题。

**精讲内容**：

这一页是 Lec 05 和 Lec 04 的"合流点"——将一维优化工具正式嵌入梯度下降框架。

回忆 Lec 04 梯度下降的更新公式：$x^{k+1} = x^k - \alpha \nabla f(x^k)$。当时我们讨论了步长 $\alpha$ 的选择：太大会发散，太小会收敛慢。那么最理想的 $\alpha$ 是什么？答案是：让下一个点的函数值尽可能小的那个 $\alpha$。

数学表述非常精确：定义辅助函数 $\phi(\alpha) = f(x^k - \alpha \nabla f(x^k))$，这是一个关于 $\alpha$ 的一元函数（因为 $x^k$ 和 $\nabla f(x^k)$ 在第 $k$ 步都是已知常量）。问题变成 $\min_{\alpha \geq 0} \phi(\alpha)$——这就是 Lec 05 讲的一维优化问题！

我们可以直接套用之前学过的方法：
1. **二分搜索**：在 $[0, \alpha_{\max}]$ 上用两个试探点不断缩小
2. **黄金分割搜索**：在 $[0, \alpha_{\max}]$ 上用 $\beta \approx 0.618$ 的缩小比
3. **基于导数**：计算 $\phi'(\alpha) = -\nabla f(x^k - \alpha \nabla f(x^k))^T \nabla f(x^k)$，用导数符号判断

这种将步长选择视为一维优化子问题的方法称为**精确线搜索**（Exact Line Search）。"精确"是因为我们试图找到真正最优的 $\alpha$。在实践中，还有一种更常用的方法是**不精确线搜索**（如 Armijo 条件、Wolfe 条件），只要求 $\alpha$ "足够好"而非最优——这在后续课程中会涉及。

精确线搜索的优点是每步下降量最大化，但缺点是每步的计算成本高（需要多次函数评估来完成一维搜索）。在实际的深度学习训练中，精确线搜索几乎不被使用——因为每次函数评估意味着一次完整的 mini-batch 前向传播，成本太高。深度学习中更常见的是固定步长（learning rate）配合 schedule 衰减，或者自适应步长方法（Adam、AdaGrad 等）。

但从理论分析的角度，精确线搜索提供了梯度下降收敛速度的上界——它是"最理想情况下"的收敛速度。这个理论基准在 Lec 07-08 的收敛性分析中会用到。

**教授口吻讲解**：

这一页特别关键——它把 Lec 05 和 Lec 04 两块拼图拼在了一起。你学了一维优化（怎么找碗底），你学了梯度下降（往哪个方向走），现在合在一起就是"往负梯度方向走，走到这个方向上的碗底"。

但我要提醒你，虽然精确线搜索理论上很完美，实际做深度学习时几乎没人用它。为什么？因为太贵了——每个 $\alpha$ 的候选值都需要一次完整的前向传播。做黄金分割搜索，哪怕只要 10 次迭代，就等于每步梯度下降要做 10 次前向传播。Adam 优化器的自适应步长虽然不是最优的，但"一步搞定"，性价比远高于精确线搜索。所以理论和实践要分开看——这是课程反复强调的一个主题。

### 第 2 页：用二分搜索选步长的具体示例

**原文翻译**：

- 假设我们处于梯度下降的第 $k$ 次迭代，位于点 $x^k$（已知）。

- 计算 $\nabla f(x^k)$（已知）。

- 用单维优化找最佳步长。

- 想一想二分搜索找最佳步长的具体过程。

[图示] 将 $f(x^k - \alpha \nabla f(x^k))$ 视为 $\alpha$ 的一元函数。在区间 $[a, b]$ 上画出关于 $\alpha$ 的函数曲线。中点为 $(a+b)/2$，$\lambda = (a+b)/2 - \varepsilon$，$\rho = (a+b)/2 + \varepsilon$。比较 $f(x^k - \lambda \nabla f(x^k))$ 与 $f(x^k - \rho \nabla f(x^k))$。

- 此外，具体函数为：

$$f(x) = 4 \ln(e^{2x_1} + e^{4x_2}) - 6x_1 - 4x_2$$

梯度为：

$$\nabla f(x) = \left(\frac{8 e^{2x_1}}{e^{2x_1} + e^{4x_2}} - 6, \quad \frac{16 e^{4x_2}}{e^{2x_1} + e^{4x_2}} - 4\right)$$

- 用二分搜索在每次迭代找最佳步长。

**精讲内容**：

这一页通过一个 log-sum-exp 类型的具体函数演示了如何将线搜索付诸实践。

函数 $f(x) = 4\ln(e^{2x_1} + e^{4x_2}) - 6x_1 - 4x_2$ 是一个经典的凸函数（log-sum-exp 的线性变换）。在 ML 中，这类函数经常出现在 softmax 回归的损失函数中——$\ln(\sum e^{z_i})$ 是 softmax 的对数归一化常数。

**具体操作流程**：

假设当前点 $x^k = (0, 0)$：

1. 计算梯度：$\nabla f(0,0) = \left(\frac{8}{2} - 6, \frac{16}{2} - 4\right) = (-2, 4)$

2. 定义一元函数：$\phi(\alpha) = f((0,0) - \alpha(-2, 4)) = f(2\alpha, -4\alpha)$

3. 在 $\alpha$ 的搜索区间 $[0, b]$（$b$ 取一个足够大的值，比如 $b = 10$）上做二分搜索：
   - 中点 $\alpha_m = 5$，取 $\lambda = 5 - \varepsilon$，$\rho = 5 + \varepsilon$
   - 比较 $\phi(\lambda)$ 和 $\phi(\rho)$，缩小区间
   - 重复直到区间足够小

这个示例很好地展示了一个关键点：虽然原问题是二维的，但每次 line search 都把它降维到了一维。维度降低让我们能用简单高效的方法（二分、黄金分割）来求解。

梯度的具体形式也值得关注：分子中的 $e^{2x_1}$ 和 $e^{4x_2}$ 与分母中的 $e^{2x_1} + e^{4x_2}$ 形成了 softmax 的形式。这意味着梯度的每个分量实际上是一个 softmax 概率乘以常数再减去一个常数——这种结构在交叉熵损失的梯度中非常常见。

**教授口吻讲解**：

这个例子虽然看起来复杂（指数函数、对数），但思路非常机械化：(1) 算梯度，(2) 把 $\alpha$ 变成一元函数，(3) 对这个一元函数做二分搜索。不管原函数多复杂，只要能算出函数值，二分搜索就能工作——这就是无导数方法的好处。

我给你一个直觉：$4\ln(e^{2x_1} + e^{4x_2})$ 这个函数的 level sets（等高线）是什么形状？因为 log-sum-exp 是 max 函数的光滑近似，所以等高线大致是"圆角矩形"——两个方向上的曲率不同。这种不均匀的曲率正是梯度下降可能 zigzag（锯齿形振荡）的原因。精确线搜索能缓解这个问题，但不能完全消除——你要等到学了 Lec 08 的强凸性才能真正理解为什么。

### 第 3 页：Constrained Optimization

**原文翻译**：

- 考虑如下形式的约束优化问题：

$$\min \quad f(x)$$
$$\text{s.t.} \quad g_1(x) \leq 0, \ldots, g_n(x) \leq 0$$
$$\quad\quad\quad h_1(x) = 0, \ldots, h_m(x) = 0$$

其中 $f, g_1, \ldots, g_n, h_1, \ldots, h_m : \mathbb{R}^n \to \mathbb{R}$。

- 在我们将考虑的所有粗糙的约束优化方法中，都会将约束优化问题转化为等价的无约束优化问题，然后用标准的梯度下降算法求解。

**精讲内容**：

这一页标志着课程从无约束优化到约束优化的**范式跳跃**。

为什么约束优化在 ML/AI 中重要？几个典型例子：

1. **SVM（支持向量机）**：$\min \frac{1}{2}\|w\|^2$ subject to $y_i(w^T x_i + b) \geq 1$
2. **概率分布约束**：优化一个概率分布 $p$，要求 $\sum p_i = 1$，$p_i \geq 0$
3. **资源分配**：最大化效用 subject to 预算约束
4. **正则化的对偶形式**：L1/L2 正则化可以等价表述为约束问题

约束分为两类：
- **不等式约束** $g_i(x) \leq 0$：定义了一个可行域，最优解可以在边界上，也可以在内部
- **等式约束** $h_j(x) = 0$：将可行域限制在一个低维流形上

本讲采用的核心策略是**化约束为无约束**——这是一种非常实用的方法论。原因很直接：我们已经有了强大的无约束优化工具（梯度下降），如果能把约束问题"翻译"成无约束问题，就能直接复用这些工具。

Slides 称这些方法为"crude"（粗糙的），因为相比 Lagrange 对偶法、KKT 条件等更精细的方法，惩罚函数法和障碍函数法概念简单、实现容易，但理论性质不如 Lagrange 方法好。然而在工程实践中，这些"粗糙"方法因为易于实现和调试而被广泛使用。

**教授口吻讲解**：

到目前为止我们一直在做无约束优化——想去哪就去哪，没有围墙。现在我们要开始处理"有围墙"的问题了。你想最小化一个函数，但不能随便走——有些地方是禁区。

怎么处理？最粗暴的方法：在禁区边界上架电网——你靠近禁区就被电击（函数值暴涨），自然就不会跑出去了。这就是惩罚函数法的思想：违反约束就加巨额罚款。方法粗糙？是的。有效？非常有效。在工业界，很多约束优化问题就是用惩罚函数法解的——因为它只需要你会做梯度下降就行，不需要学 KKT 条件那些复杂的理论。

### 第 4 页：Penalty Function Method

**原文翻译**：

- 将约束移入目标函数，对违反约束的情况施加大量惩罚。

- 原约束问题：

$$\min \quad f(x)$$
$$\text{s.t.} \quad g_1(x) \leq 0, \ldots, g_n(x) \leq 0$$
$$\quad\quad\quad h_1(x) = 0, \ldots, h_m(x) = 0$$

- 求解无约束问题：

$$\min_x \quad f(x) + \sum_{i=1}^{n} \theta_i \left(\max\{0, g_i(x)\}\right)^2 + \sum_{i=1}^{m} \gamma_i \left(h_i(x)\right)^2$$

其中 $\theta_1, \ldots, \theta_n, \gamma_1, \ldots, \gamma_m$ 是大的惩罚参数。

**精讲内容**：

这一页展示了惩罚函数法的核心公式。让我们逐项拆解：

**不等式约束的惩罚项** $\theta_i (\max\{0, g_i(x)\})^2$：
- 当 $g_i(x) \leq 0$（约束满足）时，$\max\{0, g_i(x)\} = 0$，惩罚为零
- 当 $g_i(x) > 0$（约束违反）时，$\max\{0, g_i(x)\} = g_i(x)$，惩罚为 $\theta_i g_i(x)^2$

惩罚的大小与违反程度的**平方**成正比，且被一个大系数 $\theta_i$ 放大。$\theta_i$ 越大，惩罚越重，最优解越接近可行域。

**等式约束的惩罚项** $\gamma_i (h_i(x))^2$：
- 当 $h_i(x) = 0$（约束满足）时，惩罚为零
- 当 $h_i(x) \neq 0$（约束违反）时，惩罚为 $\gamma_i h_i(x)^2$

等式约束不需要 max 操作——任何非零值都是违反。

**为什么取平方？** 这不仅仅是为了让惩罚更大。更关键的原因是**可微性**——下一页会详细讨论。$|g_i(x)|$ 在零点不可微，但 $g_i(x)^2$ 处处可微。可微性是梯度下降的前提。

**与 ML 正则化的联系**：惩罚函数法的思想和 ML 中的正则化一脉相承：
- L2 正则化 $\|w\|^2$ 就是对参数大小的二次惩罚
- 约束 $\|w\| \leq C$ 等价于在目标函数上加 $\lambda \|w\|^2$（Lagrangian 对偶）
- 惩罚参数 $\theta$ 扮演的角色与正则化系数 $\lambda$ 完全对应

这个联系不是表面的——从 Lagrange 对偶理论看，惩罚函数法可以理解为 augmented Lagrangian 的简化版本。

**教授口吻讲解**：

惩罚函数法的思想用一句话概括就是：**做错事要交罚款，罚款金额随违规程度指数增长**。$\theta_i$ 就是罚款的"底价"，$g_i(x)^2$ 是违规程度的"二次方"。为什么二次方而不是一次方？下一页详细讲，简单说就是一次方"太尖锐"，梯度下降不好处理。

你在 ML 里其实已经用过这个思想了——L2 正则化 $\lambda \|w\|^2$ 就是对参数大小的二次惩罚。约束"$\|w\|$ 不要太大"被翻译成了"$\|w\|$ 大了就在 loss 上加罚款"。惩罚函数法只是把这个思想推广到了任意约束。

### 第 5 页：为什么对不等式约束的惩罚项取平方？

**原文翻译**：

- 为什么我们对不等式约束的惩罚项取平方？

[图示——上左] 函数 $g(x)$ 的曲线，在 $a$ 和 $b$ 两个零点之间取负值（可行域），两侧取正值（不可行域）。

[图示——上右] $\max\{0, g(x)\}$ 的曲线。可行域内为 0，不可行域内等于 $g(x)$。标注：$\max\{0, g(x)\}$ 在零点处不可微。

[图示——下] $(\max\{0, g(x)\})^2 = r(x)$ 的曲线。函数变得光滑，在 $a$ 和 $b$ 处可微。

- $r'(b^-) = 0$。$r(x)$ 在 $b$ 右侧的行为类似 $g^2(x)$。

- $r'(x)$ 在 $b$ 右侧行为类似 $2g(x) \cdot g'(x)$。

- $r'(b^+) = 2g(b^+) \cdot g'(b^+) = 0$，当 $b^+$ 非常接近 $b$ 时。

**精讲内容**：

这一页回答了一个看似简单但数学上非常重要的问题：为什么要取平方？

**直接使用 $\max\{0, g(x)\}$ 的问题**：

考虑约束 $g(x) \leq 0$。惩罚项 $\max\{0, g(x)\}$ 有一个致命问题——在可行域边界（$g(x) = 0$ 的点）处**不可微**。

为什么不可微是问题？因为梯度下降需要计算梯度。如果惩罚项在关键位置不可微，梯度不存在，算法就会出错。虽然可以用次梯度（subgradient）来处理不可微点，但次梯度方法的收敛速度通常比梯度下降慢。

**取平方的效果**：

定义 $r(x) = (\max\{0, g(x)\})^2$。我们来验证它在边界点 $b$（$g(b) = 0$）处可微：

- 从左侧趋近 $b$（可行域内）：$r(x) = 0$，所以 $r'(b^-) = 0$
- 从右侧趋近 $b$（不可行域内）：$r(x) = g(x)^2$，所以 $r'(x) = 2g(x)g'(x)$

当 $x \to b^+$ 时，$g(x) \to g(b) = 0$（连续性），因此 $r'(b^+) = 2 \cdot 0 \cdot g'(b) = 0$。

左导数和右导数都等于 0，所以 $r'(b) = 0$ 存在。函数在边界点可微！

**更深的理解**：

$\max\{0, g(x)\}$ 在边界处有一个"折角"（kink），就像 ReLU 函数在零点的不可微性。取平方相当于把这个折角"磨圆"了。

这个"磨圆"的技巧在优化中非常普遍：
- Huber 损失将 L1 损失在零点附近替换为二次函数
- Softmax 是 max 函数的光滑近似
- Log-barrier（下一讲）用 $-\log(-g(x))$ 作为约束的光滑惩罚

所有这些技巧的共同目标是：在保持惩罚效果的同时获得可微性，让基于梯度的优化方法能够工作。

**教授口吻讲解**：

这个问题看起来是个小细节，但实际上反映了优化理论中一个非常核心的主题——**光滑性与效率的权衡**。

你想想 ReLU 和 sigmoid 的关系——ReLU 在零点不可微，但计算快；sigmoid 处处可微，但有梯度消失问题。这里的 $\max\{0, g\}$（像 ReLU）vs $(\max\{0, g\})^2$（像 smooth ReLU）是同样的权衡。

在实际实现中，如果你用 PyTorch 写惩罚函数法，`torch.clamp(g, min=0)**2` 就行了——PyTorch 的自动微分能正确处理这个函数。但如果你写 `torch.clamp(g, min=0)` 不取平方，在边界附近梯度的行为会不稳定，优化可能会在可行域边界上反复震荡。

### 第 6 页：惩罚函数法的完整算法与示例

**原文翻译**：

(1) 选择初始惩罚参数 $\theta_1, \ldots, \theta_n$，$\gamma_1, \ldots, \gamma_m$，容差参数 $\varepsilon$，放大倍数 $\beta > 1$，设 $k = 1$。

(2) 求解问题：

$$\min_x \left\{ f(x) + \sum_{i=1}^{n} \theta_i^k \left(\max\{0, g_i(x)\}\right)^2 + \sum_{i=1}^{m} \gamma_i^k \left(h_i(x)\right)^2 \right\}$$

设 $x^k$ 为最优解。

(3) 如果

$$\sum_{i=1}^{n} \theta_i^k \left(\max\{0, g_i(x^k)\}\right)^2 + \sum_{i=1}^{m} \gamma_i^k \left(h_i(x^k)\right)^2 \leq \varepsilon$$

则停止。否则：

$$\theta_i^{k+1} = \beta \theta_i^k, \quad i = 1, \ldots, n; \qquad \gamma_i^{k+1} = \beta \gamma_i^k, \quad i = 1, \ldots, m$$

返回步骤 2。

**示例**：

$$\min \quad (x_1 - 2)^4 + (x_1 - 2x_2)^2 \quad \text{s.t.} \quad x_1^2 - x_2 \leq 0$$

**精讲内容**：

这一页给出了惩罚函数法的完整迭代算法。核心思想是**外循环 + 内循环**的嵌套结构：

**外循环**（步骤 1-3）：控制惩罚参数的递增
- 每次外循环，惩罚参数 $\theta_i, \gamma_i$ 乘以放大系数 $\beta > 1$
- 这意味着违反约束的"代价"越来越高
- 直到惩罚项足够小（即约束近似满足）时停止

**内循环**（步骤 2 内部）：对固定的惩罚参数，用梯度下降求解无约束问题
- 这里用的就是我们已经学过的梯度下降（可以配合 line search）
- 每次外循环得到的 $x^k$ 作为下一轮内循环的初始点（warm start）

**停止条件解读**：当 $\sum \theta_i^k (\max\{0, g_i(x^k)\})^2 + \sum \gamma_i^k (h_i(x^k))^2 \leq \varepsilon$ 时停止。注意这里的条件包含了 $\theta_i^k$ 和 $\gamma_i^k$——这意味着即使违反量很小，但如果乘以已经很大的惩罚系数后仍然小，约束就被认为是足够满足了。

**示例分析**：

考虑 $\min (x_1-2)^4 + (x_1-2x_2)^2$ s.t. $x_1^2 - x_2 \leq 0$。

- 无约束最小值点：$(x_1, x_2) = (2, 1)$，$f = 0$
- 约束 $x_1^2 \leq x_2$：在点 $(2,1)$ 处，$x_1^2 = 4 > 1 = x_2$，约束违反！
- 所以无约束最优解不可行，必须在约束边界附近找到折衷

惩罚问题变为：$\min (x_1-2)^4 + (x_1-2x_2)^2 + \theta (\max\{0, x_1^2-x_2\})^2$

随着 $\theta$ 增大，解会越来越靠近约束边界 $x_1^2 = x_2$（抛物线），同时尽量接近原目标的最优方向。

**潜在问题**：惩罚参数 $\theta$ 不能无限增大。当 $\theta$ 极大时，惩罚项的 Hessian 也极大，导致无约束子问题的条件数（condition number）很差，梯度下降在子问题上收敛极慢。这就是惩罚函数法的"诅咒"——越精确，子问题越难解。Lec 07 的障碍函数法部分缓解了这个问题。

**教授口吻讲解**：

这个算法的逻辑非常直白：先轻罚，看看效果如何。不够好？加重罚款。还不够？再加。直到你乖乖遵守约束为止。

但这里有个工程上的坑：$\beta$ 不能取太大。如果 $\beta = 100$，几轮之后 $\theta$ 就是天文数字了，你的梯度下降会被惩罚项的梯度完全淹没——它只顾着满足约束，完全忘了优化目标函数。通常 $\beta = 2$ 或 $\beta = 10$ 是合理的选择。

那个示例题也很经典——$(x_1-2)^4 + (x_1-2x_2)^2$ 的无约束最小值在 $(2,1)$，但 $2^2 = 4 > 1$，违反约束。所以最优解被"推"到约束边界 $x_2 = x_1^2$ 上。这正是约束优化的精髓：最优解往往在约束边界上——这是 KKT 条件（互补松弛）的直觉来源。

## 核心收获

**收获一：精确线搜索是一维优化在梯度下降中的直接应用——但实践中性价比不高**

步长选择 $\min_\alpha f(x^k - \alpha \nabla f(x^k))$ 是一个一维优化子问题，可以直接用 Lec 05 的方法求解。这在理论分析中很有价值（提供收敛速度的上界），但在深度学习等大规模问题中，精确线搜索每步需要多次函数评估，成本过高。实际中更常用自适应步长（Adam 等）或不精确线搜索。

**收获二：惩罚函数法将约束优化化归为无约束优化——思想简洁但有数值困难**

通过在目标函数中添加约束违反的二次惩罚项，约束问题被转化为一系列无约束子问题。惩罚参数逐步增大以强制约束满足。这种方法概念简单、实现容易，但存在"精度-条件数"的两难——惩罚越大约束越满足，但子问题的求解越困难。

**收获三：可微性是梯度优化的基本要求——取平方是获得光滑性的经典技巧**

不等式约束的惩罚项取平方而非直接取绝对值，是为了保证在约束边界处的可微性。这一"磨圆"技巧在整个优化和机器学习领域普遍存在（Huber 损失、softmax、log-barrier 等），反映了一个核心权衡：光滑性 vs 精确性。
